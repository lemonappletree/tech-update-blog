# 🛠️ 2025-11-26 기술 업데이트 요약

## 🔹 Kubernetes - Kubernetes v1.35 Sneak Peek
곧 출시될 Kubernetes v1.35의 변화를 소개하는 블로그 글입니다. 이 릴리스에서는 프로젝트의 건강을 개선하기 위해 몇 가지 기능이 중단되거나 대체될 예정입니다. 주요 변경 사항으로는 cgroup v1 지원 제거, kube-proxy의 ipvs 모드 중단, containerd v1.y 지원 중단이 포함됩니다. 새로운 기능으로는 노드 선언 기능, Pod 자원 인플레이스 업데이트, Pod 인증서, taint의 수치 값 지원, 사용자 네임스페이스, OCI 이미지를 볼륨으로 마운트하는 기능이 있습니다. Kubernetes v1.35는 2025년 12월 17일에 출시될 예정이며, 자세한 정보는 릴리스 노트를 통해 확인할 수 있습니다. Kubernetes 커뮤니티에 참여하고 싶다면 여러 Special Interest Group(SIG)에 가입하거나 커뮤니티 회의에 참석할 수 있습니다.
👉 [자세히 보기](https://kubernetes.io/blog/2025/11/26/kubernetes-v1-35-sneak-peek/)

## 🔹 Spring Boot - Spring Cloud 2025.1.0 (aka Oakwood) has been released
Spring Cloud 2025.1.0(코드명 Oakwood) 릴리스가 발표되었습니다. 이 릴리스는 각 프로젝트가 5.0.0 버전으로 업데이트되었으며, Spring Framework 7과 Spring Boot 4를 기반으로 하고 있습니다. Maven Central에서 다운로드할 수 있으며, 릴리스 노트는 GitHub에서 확인할 수 있습니다. 주요 변경 사항으로는 `spring-cloud-stater-parent` 아티팩트 제거, Spring Cloud Gateway의 JSpecify 기반 null-safety 주석 추가, Spring Cloud Commons의 로드밸런서 API 버전 지원 등이 있습니다. 또한, Spring Cloud Circuitbreaker에 새로운 모듈이 추가되었으며, Resilience4j가 2.3.0으로 업그레이드되었습니다. 여러 모듈이 Java 3로 마이그레이션되었고, 다양한 네이티브 기능 지원이 개선되었습니다. 사용자는 GitHub, Gitter, Stack Overflow, Twitter 등에서 피드백을 제공할 수 있습니다. Maven과 Gradle을 통해 프로젝트에 쉽게 통합할 수 있습니다.
👉 [자세히 보기](https://spring.io/blog/2025/11/25/spring-cloud-2025-1-0-aka-oakwood-has-been-released)

## 🔹 Docker - A New Approach for Coding Agent Safety
이 기술 블로그 글에서는 Claude Code, Gemini CLI, Codex, Kiro, OpenCode와 같은 코딩 에이전트들이 개발자들의 작업 방식을 변화시키고 있다고 설명합니다. 이러한 에이전트들이 저장소를 삭제하거나 파일을 수정하고, 비밀 정보에 접근하는 등의 자율성을 갖추면서, 개발자들은 에이전트에게 유용할 만큼의 접근 권한을 부여하되, 불필요한 위험을 추가하지 않는 방법을 고민해야 하는 문제에 직면하게 됩니다. 이를 해결하기 위한 새로운 접근법이 논의됩니다.
👉 [자세히 보기](https://www.docker.com/blog/docker-sandboxes-a-new-approach-for-coding-agent-safety/)

## 🔹 Java - ONNX Based Generative AI LLMs in Java with Project Babylon
블로그 글 요약: Open Neural Network Exchange (ONNX)는 머신러닝 모델을 다양한 플랫폼에서 배포할 수 있는 범용 포맷을 제공합니다. 전통적으로 대규모 언어 모델(LLM)은 Python에서 PyTorch, TensorFlow, scikit-learn 등의 프레임워크를 사용해 개발된 후 ONNX로 변환되어 실행됩니다. 이 프레젠테이션에서는 인공지능 모델링에 흔히 사용되지 않는 Java를 활용하여 ONNX 모델을 생성하는 방법을 설명합니다. 또한, Project Babylon의 코드 반영 기능을 활용한 ONNX 기반 생성 AI LLMs의 개념을 탐구합니다. 프레젠테이션은 Java로 작성된 LLM을 ONNX 포맷으로 변환하고 실행하는 실질적인 예시를 제공합니다.
👉 [자세히 보기](https://inside.java/2025/11/25/devoxxbelgium-java-generative-ai/)
