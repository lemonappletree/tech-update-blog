# 🛠️ 2025-12-04 기술 업데이트 요약

## 🔹 Kubernetes - Kubernetes v1.35 Sneak Peek
Kubernetes v1.35의 출시가 다가오면서 프로젝트는 계속 발전하고 있습니다. 이 블로그 글에서는 v1.35 출시에서 계획된 변경 사항을 설명하며, 이를 통해 Kubernetes 클러스터의 원활한 운영을 보장하고 최신 개발 정보를 제공합니다. 주요 변경 사항으로는 cgroup v1 지원 중단, kube-proxy에서 ipvs 모드의 폐기, containerd v1.y 지원 종료 등이 있습니다. 또한, 노드 선언 기능, Pod 자원 인플레이스 업데이트, Pod 인증서, taint의 수치적 값, 사용자 네임스페이스, OCI 이미지를 볼륨으로 마운트하는 기능 등의 새로운 기능이 소개될 예정입니다. Kubernetes v1.35는 2025년 12월 17일에 출시될 계획입니다.
👉 [자세히 보기](https://kubernetes.io/blog/2025/11/26/kubernetes-v1-35-sneak-peek/)

## 🔹 Spring Boot - This Week in Spring - December 2nd, 2025
이번 주 "This Week in Spring" 블로그 글에서는 실수로 이전 콘텐츠를 게시했다가, 이를 수정하는 과정에서 초안을 삭제하는 실수를 저지른 이야기를 전하며, 충분한 수면의 중요성을 강조하고 있습니다. 이번 주에는 Spring Tools 프로젝트의 차세대 버전, Spring Cloud 2025.1.0 릴리스, Spring Data AOT 저장소 지원, Spring AI의 AWS re:Invent 참여, Spring Framework 7과 Spring Boot 4의 새로운 기능에 대한 논의 등이 포함됩니다. 또한, Spring AI와 관련된 여러 흥미로운 기사와 Spring Boot 4로의 마이그레이션, Spring AI의 OpenAI SDK 지원 등 다양한 주제를 다루고 있습니다.
👉 [자세히 보기](https://spring.io/blog/2025/12/02/this-week-in-spring-december-2nd-2025)

## 🔹 Docker - Securing the Docker MCP Catalog: Commit Pinning, Agentic Auditing, and Publisher Trust Levels
이 기술 블로그 글은 Docker MCP 카탈로그의 보안을 강화하는 방법에 대해 설명합니다. AI 도우미를 실제 도구에 연결할 때 신뢰가 가장 중요하며, MCP 컨테이너화는 격리와 보안성을 제공하지만, 악성 코드 노출을 줄이기 위해 Docker MCP 솔루션의 신뢰성과 보안을 계속 강화하고 있다고 합니다. 이를 위해 커밋 고정, 에이전틱 감사를 실시하고 발행자 신뢰 수준을 설정하는 방법을 사용하고 있습니다. MCP 생태계가 확장됨에 따라 이러한 보안 조치는 더욱 중요해지고 있습니다.
👉 [자세히 보기](https://www.docker.com/blog/enhancing-mcp-trust-with-the-docker-mcp-catalog/)

## 🔹 Java - So Long and Thanks for All the Applets
제목: "So Long and Thanks for All the Applets"

👉 [자세히 보기](https://inside.java/2025/12/03/applet-removal/)
